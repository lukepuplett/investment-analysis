GITLAB DUO AGENT PLATFORM (DAP)
Launch: January 2026 (7 weeks into market)

Three Core Pillars:
1. Workflows: Unified place where teams and AI agents collaborate
2. Context: Rich semantic access to full SDLC data
3. Guardrails: Deploy anywhere with security/compliance embedded

Key Capabilities:
- Orchestration platform for AI agents across software life cycle
- Same context, permissions, security model as existing GitLab platform
- AI at point of execution (where code lives, merged, built, deployed)
- Reduces friction for agents and humans managing them

Usage-Based Pricing Model:
- Customers pay for agent work where every engineer can delegate tasks to multiple agents
- Revenue grows with value delivered
- Complements seat-based model

QUESTION: Security Differentiation vs Claude Code Security (Koji Ikeda, BofA)

Bill Staples Answer:
"Really comes down to difference between suggestions and certification."
- Claude Code Security: helps developers write better code at authoring time
- GitLab: independent system certifying code ready for production
- Developers can ignore Claude's recommendations
- Can't bypass GitLab's execution policy for pipeline, merge request approval rules
- Companies ship what is secure and meets engineering standards
- These are complementary: Claude improves code at authoring, GitLab governs shipping

QUESTION: DAP Pricing Model (Sanjit Singh, Morgan Stanley)

Bill Staples Answer:
- Competitors have more than just seat-based pricing (they throttle or charge overages)
- GitLab DAP pricing model:
  * Every Premium seat gets $12 in credits included
  * Every Ultimate seat gets $24 in credits included
  * Want customers to win hearts/minds without new contracts
  * If finding value, can opt into on-demand credits
  * On-demand credits: pay-as-you-go at ~$1/credit
  * Creates demand signal for sales (see usage, have conversation)
  * Offer additional discounts for committed credits as monthly minimum
  * Monthly minimums turn into subscription recognized ratably (drives flywheel)
- Model advantages:
  * Better value for customers vs throttled/overage model
  * More efficient
- Competitive observation:
  * Existing tools "heavily subsidized by venture capitalists"
  * Prediction: "This year you'll see many enterprise tools begin to move to API-based charges vs seat-based, which could raise bills for enterprises"

QUESTION: Legacy Code Modernization & TAM Expansion (Kingsley Crane, Canaccord)

Bill Staples Answer:
- Some software can theoretically be rewritten
- Exciting experiments with code generation in closed loops
- More difficult for complex legacy applications (mainframes, enterprise software with integrations)
  * Context often in people's heads, not documented
  * Years of integration assumptions
  * Hard to discover purely through code

Three Modes Framework (Parallels to Cloud Migration):
1. Mode 1 - No Agent Touch
   * Mission-critical software for businesses
   * Especially financial/regulated and public sector
   * Agents not allowed (security, privacy, sensitivity)

2. Mode 2 - Human-Agent Collaboration (Current - DAP Supports)
   * Brownfield codebases
   * Orchestration patterns emerging
   * Will continue for some time

3. Mode 3 - Increasingly Automated
   * Better orchestration and closed-loop iteration
   * Especially greenfield projects
   * Less human involvement except steering/orchestrating
   * Agents handling more autonomously
